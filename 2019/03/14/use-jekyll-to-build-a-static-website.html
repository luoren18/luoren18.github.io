使用Jekyll搭建一个个人博客

REN 14 Mar 2019

Jekyll搭建个人博客

  1. 初始化一个Jekyll 项目

    jekyll new myblog

  2. 安装依赖

    cd myblog
bundle install

  3. 运行当前博客

    bundle exec jekyll server
# 或者
jekyll server

  4. 编译成 markdown 文件成静态Html 代码,编译好的文件将放在_site 文件夹中

    jekyll build
# 或者
jekyll b
# 或者自定义编译路径
jekyll build --Destination=./dest

  5. 使用主题

    从GitHub上找到相关主题，下载 dactl

    git clone https://github.com/melangue/dactl.git
cd dactl
bundle install
jekyll server

  6. 使用dactl 主题在_posts下自己编写博客

    ---
layout: post
title:  "免费搭建一个基于Jekyll的静态博客"
tags:
  - blog
  - jekyll
hero: https://source.unsplash.com/collection/430471/
overlay: blue
published: true
---
定义博客的标题，标签，页面颜色
编写博客的内容

  7. 在GitHub上部署博客并且绑定自己的域名

    # 删除从 GitHub上下载下来的博客中的.git 文件夹
rm -rf .git
# 自己生成一个本地的git 项目
git init
# 将博客项目文件添加到git管理
git add -A .
# 将项目提交到本地的git仓库
git commit -m 'init'
# 绑定远程的git 仓库
git remote add origin https://github.com/luoren18/2345.git
# 将代码push到远程的git 仓库
git push -u origin master

    打开设置，跳转到GitHub pages选项，绑定自己的域名，至此就可以通过自己的域名访问GitHub上的博客了。

  8. 使用surge 部署博客

    需要安装nodeJS 的环境

     npm install --global surge
 surge

  9. 使用插件博客变得更加完善

    • 安装插件

      安装Jekyll-admin 插件，可以对Jekyll进行后台管理。在项目的根路径下的Gemfile 文件中添加 gem "jekyll-admin" ，然后运行 bundle install 安装依赖，运行Jekyllbundle exec jekyll server，访问http://localhost:4000/admin。

    • 使用插件

      在插件的Github 主页的 作者一般都会说明如何使用当前插件，仔细阅读即可。

    • 寻找插件

      Jekyll Plugins

      http://www.jekyll-plugins.com/
